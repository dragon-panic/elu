## Scope

Implement `elu init --from <path>`: infer a starter package manifest by walking a project tree and recognizing well-known files (`Cargo.toml`, `package.json`, `pyproject.toml`, etc.).

Stub: `cmd/init.rs:10`. PRD: `docs/prd/cli.md:119-141`.

## Slice

Pure CLI/heuristics — no lower-ring change needed.

1. **Red** — integration test: given a fixture directory containing `Cargo.toml`, `elu init --from <fixture>` writes a manifest naming the rust toolchain hook (or whatever the PRD specifies) and the inferred package name/version.
2. **Green** — wire a small inference function in \`elu-cli\` that maps recognized files → manifest stanzas.
3. Cover at least: rust (Cargo.toml), node (package.json), python (pyproject.toml). One file = one match. Conflict between two recognized files = error with a clear message.

## Out of scope

- Multi-language merging / smart hook composition.
- Parsing any of the recognized files for transitive deps — just identify and pick the right ecosystem hook.
